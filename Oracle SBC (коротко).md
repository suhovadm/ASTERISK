Oracle SBC.  
Oracle Communications Session Border Controller.  
Это система, которая стоит на границе VoIP-сети и управляет SIP-трафиком между разными сетями.  

Простыми словами.  
Oracle SBC – это «телеком-шлюз безопасности», который:  
•	соединяет вашу VoIP-сеть с внешними операторами,  
•	защищает SIP-инфраструктуру,  
•	контролирует и фильтрует звонки.  

Основные функции.  

Безопасность:  
•	защита от SIP-атак – флуд (DDoS/спам, т.е. перегруз системы), фрод, сканирование,  
•	скрытие внутренней сети (topology hiding),  
•	контроль доступа.  

Совместимость.  
•	Исправляет «кривой SIP» от разных операторов,  
•	нормализация SIP-сигналов,  
•	трансформация заголовков SIP.  

Маршрутизация.  
•	Выбор маршрута для звонка,  
•	failover на резервных операторов,  
•	балансировка нагрузки.  

Медиа-функции.  
•	Транскодинг (разные аудиокодеки).  
•	Управление RTP-трафиком. RTP – это Real-Time Protocol.  

Где используется?  
•	У телеком-операторов.  
•	В крупных корпоративных VoIP-сетях.  
•	В контакт-центрах.  
•	В облачных коммуникационных платформах.  

Как выглядит архитектура?  
Пример схемы:  
IP-телефоны / Asterisk / FreeSWITCH / Kamailio.  
|  
Oracle SBC.  
|  
операторы связи / PSTN / другие VoIP-сети.  

Чем отличается от Kamailio?  
•	Kamailio – программный SIP-прокси (маршрутизация, гибкость, масштабирование).  
Условно говоря, это Linux + iptables.  
•	Oracle SBC – enterprise-уровень, «всё в одном» (безопасность + медиа + совместимость).  
А это, условно готовый роутер + firewall.
